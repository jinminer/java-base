# 泛型

## 不要使用原始类型

* 使用原始类型可能导致运行时异常
* 原始类型仅用于与泛型引入之前的传统代码的兼容性和互操作性。



## 消除非检查警告



## 列表优于数组

* 不可具体化的类型`（nonreifiable types）[JLS，4.7]`。 
  * 不可具体化的类型是其运行时表示包含的信息少于其编译时表示的类型。
  *  由于擦除，可唯一确定的参数化类型是无限定通配符类型，如 List<?> 和 Map<?, ?> （条目 26）。
  * 尽管很少有用，创建无限定通配符类型的数组是合法的。
* 数组是协变和具体化的; 
  * 数组提供运行时类型的安全性，但不提供编译时类型的安全性
* 泛型是不变的，类型擦除的。
